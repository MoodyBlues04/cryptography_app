How AES Works

AES operates on a 4x4 matrix of bytes called the state. The algorithm consists of multiple rounds of transformations, depending on the key size:

  • 10 rounds for 128-bit keys
  • 12 rounds for 192-bit keys
  • 14 rounds for 256-bit keys

Each round (except the last) consists of four transformations:

  1. SubBytes: Substitutes each byte in the state using a substitution box (S-box).
  2. ShiftRows: Shifts the rows of the state matrix.
  3. MixColumns: Mixes the columns of the state matrix using matrix multiplication in Galois Field (GF(2^8)).
  4. AddRoundKey: XORs the state with a round key derived from the main key.

The final round skips the MixColumns step.

Key Expansion

AES uses a key schedule to generate round keys from the main encryption key. The key expansion process involves:

  1. Dividing the key into 4-byte words.
  2. Applying a series of transformations (rotation, S-box substitution, and XOR with round constants) to generate new words.
  3. Combining the words to form the round keys.

The number of round keys depends on the number of rounds. For example, AES-128 requires 11 round keys (1 initial key + 10 round keys).

Detailed Steps of AES

1. SubBytes

Each byte in the state matrix is replaced with a corresponding byte from the S-box. The S-box is a lookup table that provides non-linear substitution.

2. ShiftRows

The rows of the state matrix are shifted cyclically:

  • Row 0: No shift
  • Row 1: Shifted left by 1 byte
  • Row 2: Shifted left by 2 bytes
  • Row 3: Shifted left by 3 bytes

3. MixColumns

Each column of the state matrix is transformed using matrix multiplication in GF(2^8). This step mixes the bytes within each column.

4. AddRoundKey

The state matrix is XORed with the round key. This step adds the key material to the state.

Security and Applications

AES is considered highly secure and is used in various applications, including:

  • Secure communication protocols (e.g., TLS, SSL)
  • File and disk encryption (e.g., BitLocker, FileVault)
  • Data protection in databases and cloud storage
